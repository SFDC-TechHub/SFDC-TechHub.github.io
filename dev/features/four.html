Lightning Components: Synchronize Component Data Without a Page Refresh Using RefreshView API (GA)
Contributed by : Subhojeet Ghosh

RefreshView API supports refreshes that are triggered by end users or web components, updates the data for a specific hierarchy of components, known as a view, without reloading an entire page. LWC lacked the data refresh API and could only refresh using an Aura wrapper and the legacy "force:refreshView" which was not meeting the requirements of modern web development. The RefreshView API and lightning/refresh module provide a standard way to refresh component data in custom LWC and Aura components. This feature is Generally Available (GA) now from this release.

Important points to note for this feature:

  • Lightning Web Security (LWS) must be enabled in the Salesforce org.
  • RefreshView API supports refreshes that are triggered by end users or web components.
  • The base Lightning Aura components currently don’t support RefreshView API.
  • Lightning Locker doesn’t support RefreshView API.
