Lightning Components: Base Lightning Component Internal DOM Structure Changes for Future Native Shadow Support
Contributed by : Subhojeet Ghosh

Salesforce is working to align the base Lightning components with Web Components Standards by preparing them to support native shadow DOM. These updates change the internal DOM structure of these base Lightning components.

Advantages:

  • To enhance the performance of these base lightning components.
  • To comply with Web Components Standards.

Which all base Lightning components have been updated so far ?

  • In Spring '23, the internal structure of 
    lightning-button
    and 
    lightning-button-group
    were changed.
  • In Summer '23, following components have been adapted: 
    1. lightning-dual-listbox
    2. lightning-dynamic-icon
    3. lightning-breadcrumb
    4. lightning-breadcrumbs
    5. lightning-vertical-navigation
    6. lightning-vertical-navigation-section
    7. lightning-vertical-navigation-item
    8. lightning-vertical-navigation-overflow
    9. lightning-vertical-navigation-item-badge
    10. lightning-vertical-navigation-item-icon

Please make sure your tests are not relying on the protected internal DOM structure for the above components. If it does, rewrite them ASAP.
