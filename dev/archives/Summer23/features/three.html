Lightning Components: Attribute Names Support More Patterns
Contributed by : Subhojeet Ghosh

LWC attribute names now support more patterns. This change applies to custom LWC in Lightning Experience, Experience Builder sites, and all versions of the Salesforce mobile app.

Supported patterns for attribute names are:

  • Attribute names can now start with an 
    underscore (_)
    .
  • Attribute names can now start with a 
    dollar sign ($)
    .
  • Attribute names can now start with a lowercase alphabetic character.
  • Attribute names can now start with an 
    optional hyphen followed by an alphabetical character (-a)
    .
  • Attribute names can include 
    duplicate underscores (__)
    .
  • Attribute names can include an 
    underscore followed by a hyphen(_-)
    .
  • Attribute names can include a 
    hyphen followed by an underscore (-_)
    .
  • Attribute names must still end with an alphanumeric character.
