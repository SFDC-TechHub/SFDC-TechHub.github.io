Lightning Components: Enable Third-Party Integrations with Light DOM (GA)
Contributed by : Subhojeet Ghosh

LWC render in shadow DOM by default. This provides a strong encapsulation but poses challenges in global styling and many 3rd party integrations. In this release, Light DOM is now generally available (GA).

Advantages of Light DOM:

  • With Light DOM, the components are now attached to the host element instead of its shadow tree.
  • This facilitates global styling. Therefore custom branding can be applied easily to the components & its child components.
  • This allows standard browser query APIs like 
    "querySelector"
    & 
    "querySelectorAll"
    to traverse the DOM. No need to traverse the shadow root. Therefore it provides simple integration with third-party libraries and tools.
  • Events bubble through components even if 
    "composed"
    is 
    "false"
    in light DOM, since there's no shadow root. With light DOM, events arenâ€™t retargeted i.e, 
    "event.target"
    returns the button that triggered the event, instead of the containing component.

Supported:

  • A Light DOM component can contain a shadow DOM component.
  • a shadow DOM component can also contain a light DOM component.
  • In light DOM, you can use scoped styles 
    (*.scoped.css)
    to apply CSS to elements on the component only. This behavior is similar to style encapsulation with shadow DOM.

Not Supported:

  • Light DOM cannot be restricted to specific namespaces.
  • Base components are always rendered in shadow DOM.
  • Aura components can't use light DOM. However, an Aura component can contain an LWC component that uses light DOM.
